# Админка для сайта-витрины услуг по БПЛА

Этот документ описывает, как запускать и использовать админ-панель для управления данными сайта (услуги, портфолио и т.п.).

## 1. Состав админки

- **API админки** — Node.js + Express, крутится локально на `http://localhost:4000`, читает и пишет файлы в папке `data/`.
- **Интерфейс админки** — SPA на Vue + Vite, крутится локально на `http://localhost:5173` (порт может отличаться) и обращается к API по `http://localhost:4000/...`.

Публичный сайт (витрина) по-прежнему развёрнут как статический сайт на GitHub Pages и работает независимо от админки.

## 2. Запуск API для админки

В корне репозитория `bplav1`:

```bash
npm run admin-api
```

Ожидаемое сообщение в консоли:

```text
Admin API listening on http://localhost:4000
```

Это локальный сервер, который:

- отдаёт текущие данные (например, список услуг) из `data/services.json` по GET-запросам;
- принимает изменения и перезаписывает JSON-файл по POST-запросам.

Терминал с `npm run admin-api` должен оставаться запущенным, пока вы работаете с админкой.

## 3. Запуск интерфейса админки

В отдельном терминале:

```bash
cd admin-panel
npm run dev
```

После старта Vite выведет URL вида:

```text
Local: http://localhost:5173/
```

(если порт 5173 занят, будет другой.)

Дальше:

- Открыть указанный URL в браузере (в Codespaces — через кнопку «Open in browser» или аналогичную).
- На главной странице отображается интерфейс админки.

## 4. Работа с услугами

В текущей версии админка управляет списком услуг, который хранится в файле:

```text
data/services.json
```

Логика работы:

- При открытии страницы админка делает запрос `GET http://localhost:4000/api/services` и получает массив объектов услуг.
- На экране отображается:
  - список существующих услуг;
  - форма добавления/редактирования услуги.

Поддерживаемые операции:

- **Добавить услугу**  
  Заполнить форму «Новая услуга» и нажать «Сохранить».  
  Внутри админка:
  - добавляет новую запись в массив;
  - отправляет обновлённый массив услуг POST-запросом на `http://localhost:4000/api/services`;
  - API перезаписывает `data/services.json`.

- **Редактировать услугу**  
  Нажать кнопку «Редактировать» рядом с нужной услугой, изменить поля в форме и нажать «Сохранить».  
  Внутри:
  - услуга с соответствующим `id` обновляется;
  - весь массив отправляется POST-запросом на API и сохраняется в `data/services.json`.

- **Удалить услугу**  
  Нажать кнопку «Удалить», подтвердить действие.  
  Внутри:
  - услуга удаляется из массива;
  - обновлённый массив отправляется POST-запросом на API и сохраняется в `data/services.json`.

Все изменения происходят локально, с использованием файлов в папке `data/`.

## 5. Обновление публичного сайта после изменений

После того как данные изменены через админку, публичный сайт нужно пересобрать и заново задеплоить на GitHub Pages.

Действия в корне `bplav1`:

```bash
npm run generate
npm run deploy
```

- `npm run generate` — запускает генератор, который читает данные из `data/` и собирает статический сайт в папку `dist/`.
- `npm run deploy` — публикует содержимое папки `dist/` на GitHub Pages (ветка `gh-pages`), после чего сайт обновляется по адресу:

```text
https://buhtig-sudo-azar.github.io/bplav1/
```

Проверять обновление лучше с жёстким обновлением страницы (Ctrl+F5 / Ctrl+Shift+R) или в режиме инкогнито, чтобы избежать кеша браузера.

## 6. Остановка админки

- Остановить API: в терминале с `npm run admin-api` нажать `Ctrl+C`.
- Остановить интерфейс админки: в терминале с `npm run dev` (в папке `admin-panel`) нажать `Ctrl+C`.

Публичный сайт для клиентов при этом продолжает работать, так как он уже задеплоен как статический контент на GitHub Pages.
